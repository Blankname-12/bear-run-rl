# S4-02 奖励设计五轮迭代 —— 完整记录

## 迭代零：背景

预训练行为克隆模型（SimpleCNN，val_acc ~82%）能在游戏中躲避 5+ 障碍物。PPO 的目标是在此基础上微调，通过在线试错进一步提升性能。奖励函数是连接"预训练知识"和"在线探索"的核心桥梁。

---

## 迭代一：教师蒸馏 + 重惩罚

### 配置

```python
REWARD_TEACHER_MATCH    = +1.0   # agent 动作与冻结教师模型一致
REWARD_TEACHER_MISMATCH = -0.5   # agent 动作与教师不一致
REWARD_ALIVE            = +0.1   # 每步存活（无教师时的兜底）
REWARD_DEATH            = -10.0  # 死亡
REWARD_HEART_LOST       = -1.0   # 红心消失（受伤）
```

### 代码逻辑

```python
teacher_action = self._teacher_predict(raw)
if action_idx == teacher_action:
    step_reward = +1.0
else:
    step_reward = -0.5
reward = step_reward + heart_lost_penalty
```

### 观察到的行为

训练 40 分钟后，agent **只输出 noop（动作 4）**：

```
[Step 18-38] 动作=不动 奖励=+0.00 (noop=0)
[Step 39]    死亡 奖励=-10.00
```

### 失败原因分析

1. **教师不准**：由于 PPO 网络和教师网络输入格式不同（见 S4-01），教师预测对 agent 是噪声
2. **noop 正反馈循环**：
   - 标注数据中 noop 占多数（~50-70%）
   - 教师学会了大量预测 noop
   - Agent 选 noop → 教师也预测 noop → 匹配！→ 得 +1.0
   - Agent 选其他动作 → 教师大概率不匹配 → 扣 -0.5
   - **reinforcement**: noop = +1.0, 动作 = -0.5
3. **死亡惩罚放大**：偶尔做动作导致死亡 → -10.0，更强化了"不动最安全"的信念

**关键洞察**：在奖励函数中，noop 是唯一能稳定获得正奖励的动作。策略坍缩到 noop 是理性的。

## 迭代二：noop 不给分

### 配置变化

```python
# 在 step() 中硬编码
if action_idx == 4:    # noop
    step_reward = 0.0  # 覆盖教师奖励
    match_str = "(noop=0)"
```

其余不变：MISMATCH 仍为 -0.5。

### 观察到的行为

从"全 noop"变成"全滑铲"：

```
[Step 17-22] 动作=滑铲 奖励=+0.00 ≠教师(左移)    ← 连续 6 步滑铲
[Step 23]    动作=滑铲 奖励=+1.00 =教师           ← 偶尔匹配
[Step 24-28] 动作=滑铲 奖励=+0.00 ≠教师(不动)    ← 继续滑铲
[Step 29]    动作=不动 奖励=+0.00 (noop=0)        ← 偶尔 noop
[Step 30-38] 动作=滑铲 奖励=+0.00                 ← 继续滑铲
```

### 失败原因分析

1. 教师不准 → agent 几乎匹配不到教师 → 所有非 noop 动作都拿 0 分
2. noop 也拿 0 分
3. **所有动作奖励完全相同** → agent 无法区分好坏
4. 策略随机收敛到最先被采样的动作（这次碰巧是滑铲）

**关键洞察**：noop=0 去掉了一个坏的局部最优，但没有创建任何好的梯度方向。策略会坍缩到某个随机动作。

## 迭代三：去掉不匹配惩罚

### 配置变化

```python
REWARD_TEACHER_MISMATCH = 0.0  # 从 -0.5 改为 0
```

### 奖励结构

| 条件 | 奖励 |
|---|---|
| noop | 0 |
| 动作匹配教师 | +1.0 |
| 动作不匹配教师 | **0**（不再扣分） |
| 死亡 | -10.0 |

### 观察到的行为

仍坍缩到 noop（日志略，与迭代一类似）。

### 失败原因分析

- 做动作匹配教师 → +1.0（但匹配不到，因教师不准）
- 做动作不匹配 → 0（和 noop 一样）
- 做动作可能导致死亡 → -10.0
- noop → 0，且"更安全"（至少不会因为乱动而撞到障碍物）

**策略权衡**：动作 = 0 分 + 死亡风险，noop = 0 分 + 无额外风险。Agent 理性选择 noop。

**关键洞察**：正奖励不够（匹配不到），负奖励不对称（死亡惩罚只惩罚动作不惩罚 noop）。即使 noop 也最终会死，但 agent 无法建立"noop → 死亡"的因果链（因为死亡回溯覆盖的是 noop 帧，而 noop 帧本身 0 分）。

## 迭代四：去掉教师，纯环境奖励

### 配置

完全移除教师模型。奖励仅依赖环境信号：

```python
# 无教师时的逻辑
if action_idx == 4:
    step_reward = 0.0       # noop 不给分
else:
    step_reward = +0.1      # 任何非 noop 动作 +0.1

REWARD_DEATH = -10.0         # 死亡
REWARD_HEART_LOST = -1.0    # 红心丢失
```

### 训练日志（第 27-29 局）

```
第 27 局 — 存活 35 步 | 奖励: -11.00 | Avg20: -10.36
第 28 局 — 存活 39 步 | 奖励: -11.00 | Avg20: -10.47
第 29 局 — 存活 32 步 | 奖励: -11.00 | Avg20: -10.57
```

所有局奖励都是 -11.00（= -10 死亡 + -1 红心丢失），平均奖励稳定在 -10 左右。

### 失败原因分析

1. **动作间无区分度**：跳、滑、左、右全部都 +0.1，agent 无法知道哪个好哪个坏
2. **信用分配失效**：
   - 存活 35 步，其中 ~17 步是暖机（等红心），~18 步实际游戏
   - 死亡 -10.0 通过 GAE（γ=0.99, λ=0.95）回溯分配
   - 18 步中，正确的动作（躲障碍）和错误的动作（撞障碍）**拿到完全相同的 +0.1**
   - GAE 把 -10 均匀洒在最近的十几帧上，无法区分哪些动作导致了死亡
3. **死亡回溯也在惩罚 noop**：第 12-15 帧前的回溯惩罚同样打在 noop 帧上，agent 学到的是"无论做什么都会死"

**关键洞察**：纯环境奖励对单帧分类任务是灾难性的。没有逐帧的"这个动作对不对"信号，只有稀疏的最终结果信号。PPO 需要大量样本和极佳的价值函数才能从稀疏奖励中学习，而这在 128×128 像素级别的状态空间中几乎不可能。

## 迭代五（当前）：教师蒸馏 + 轻惩罚 + noop 小罚

### 设计原则

经过四轮失败，总结出以下原则：

1. **教师是唯一可靠的动作级别信号**：预训练模型在每帧都能给出"应该做什么"的建议。这是唯一能提供逐帧区分度的信号源。
2. **惩罚要轻**：重惩罚（-10）让 agent 恐惧所有非 noop 动作，扼杀探索。惩罚应该只够引导，不够威慑。
3. **noop 要轻微惩罚**：不给分不够（无梯度方向），需要微弱的负信号让 agent 偏好动作。
4. **教师只给糖不给鞭子**：匹配 +1.0（好！），不匹配 0（没关系），不惩罚偏离教师的行为。
5. **前提**：输入格式已对齐（S4-01），教师预测对 agent 有直接参考价值。

### 最终配置

```python
# config.py
REWARD_TEACHER_MATCH    = 1.0   # agent 动作与教师一致 → 强正信号
REWARD_TEACHER_MISMATCH = 0.0   # 不一致不惩罚（鼓励探索）
REWARD_DEATH            = -1.0  # 死亡轻罚（原 -10.0）
REWARD_HEART_LOST       = -1.0  # 红心丢失（当前不生效，已从 reward 计算中移除）
```

### 代码逻辑

```python
# train_ppo.py RLGameEnv.step()
if action_idx == 4:
    step_reward = -0.1              # noop 小惩罚
    match_str = "(noop)"
elif self._teacher is not None:
    teacher_action = self._teacher_predict(raw)  # 独立冻结模型
    if action_idx == teacher_action:
        step_reward = 1.0           # 匹配教师
        match_str = "=教师"
    else:
        step_reward = 0.0           # 不匹配不罚
        match_str = f"≠教师({教师预测动作})"
else:
    step_reward = 0.1               # 无教师时的兜底
    match_str = ""

reward = step_reward                # 不再叠加红心丢失扣分
```

### 预期效果

因为输入格式统一后 agent 和教师架构完全一致，权重 1:1 加载：
- agent 初始策略 ≈ 教师策略
- agent 每一步与教师预测一致的概率 > 80%
- 大部分步骤 agent 拿到 +1.0
- agent 偶尔不匹配教师 → 0 分（不扣）
- agent 做 noop → -0.1（小惩罚）
- agent 死亡 → -1.0 + 回溯 -4.0 = -5.0

**正奖励路**: 匹配教师 → +1.0 → 价值上升 → 策略强化匹配行为
**探索路**: 不匹配教师 → 0 分 → 如果探索到更好的动作（存活更长），死亡惩罚减少 → 自然强化
**防坍缩**: noop → -0.1 → 持续 noop 会累积负奖励，agent 必须做动作

### 死亡回溯惩罚

```python
# 死亡时，对第 12~15 帧前施加额外惩罚
# 设计逻辑：用户观察到"死亡是一段操作的结果"，前 11 帧无辜，12-15 帧是罪魁祸首
if done and len(agent.buffer.rewards) >= 16:
    for i in range(13, 17):  # i=13,14,15,16 → 即第 12,13,14,15 帧前
        if i <= len(agent.buffer.rewards):
            agent.buffer.rewards[-i] += config.REWARD_DEATH  # -1.0 each
    # 总计额外 -4.0，加上死亡时刻的 -1.0 = -5.0
```
