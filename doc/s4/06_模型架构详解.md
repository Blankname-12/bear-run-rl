# S4-06 模型架构与权重映射详解

## 1. SimpleCNN（行为克隆预训练）

### 1.1 完整架构

```
输入: (B, 3, 128, 128)  # 单帧 RGB，值域 [0,1]

ConvBlock #1: Conv2d(3, 32, 3×3, pad=1) → ReLU → MaxPool2d(2)
  输出: (B, 32, 64, 64)

ConvBlock #2: Conv2d(32, 64, 3×3, pad=1) → ReLU → MaxPool2d(2)
  输出: (B, 64, 32, 32)

ConvBlock #3: Conv2d(64, 128, 3×3, pad=1) → ReLU → MaxPool2d(2)
  输出: (B, 128, 16, 16)

ConvBlock #4: Conv2d(128, 256, 3×3, pad=1) → ReLU → MaxPool2d(2)
  输出: (B, 256, 8, 8)

AdaptiveAvgPool2d(4, 4)
  输出: (B, 256, 4, 4) = 4096 维

Flatten → Linear(4096, 256) → ReLU → Dropout(0.3) → Linear(256, 6)
  输出: (B, 6) logits
```

总参数：约 4.3M（4,333,382）
- Conv1: 3×32×9 + 32 = 896
- Conv2: 32×64×9 + 64 = 18,496
- Conv3: 64×128×9 + 128 = 73,856
- Conv4: 128×256×9 + 256 = 295,168
- FC1: 4096×256 + 256 = 1,048,832
- FC2: 256×6 + 6 = 1,542
- 总计 ≈ 4.3M

### 1.2 权重初始化

```python
def _init_weights(self):
    for m in self.modules():
        if isinstance(m, nn.Conv2d):
            nn.init.kaiming_normal_(m.weight, mode="fan_out")
            nn.init.constant_(m.bias, 0)
        elif isinstance(m, nn.Linear):
            nn.init.normal_(m.weight, mean=0, std=0.01)
            nn.init.constant_(m.bias, 0)
```

### 1.3 PyTorch Sequential 命名规则

```python
self.classifier = nn.Sequential(
    nn.Flatten(),              # classifier.0
    nn.Linear(4096, 256),     # classifier.1
    nn.ReLU(inplace=True),    # classifier.2
    nn.Dropout(0.3),          # classifier.3
    nn.Linear(256, 6),        # classifier.4
)
```

## 2. ActorCriticCNN（PPO 强化学习）

### 2.1 完整架构

```
输入: (B, 3, 128, 128)  # 与 SimpleCNN 完全一致

=== 共享骨干（与 SimpleCNN 完全相同）===
ConvBlock #1: Conv2d(3, 32, 3×3) → (B, 32, 64, 64)
ConvBlock #2: Conv2d(32, 64, 3×3) → (B, 64, 32, 32)
ConvBlock #3: Conv2d(64, 128, 3×3) → (B, 128, 16, 16)
ConvBlock #4: Conv2d(128, 256, 3×3) → (B, 256, 8, 8)
AdaptiveAvgPool2d(4, 4) → (B, 256, 4, 4)
Flatten → (B, 4096)

=== Actor 头（策略）===
Linear(4096, 256) → ReLU → Linear(256, 6) → (B, 6) logits

=== Critic 头（价值）===
Linear(4096, 256) → ReLU → Linear(256, 1) → (B, 1) value
```

总参数：约 4.4M

### 2.2 与 SimpleCNN 的结构对应关系

```
SimpleCNN                    ActorCriticCNN
─────────────────────────────────────────────────
conv1  (ConvBlock)           conv1  (ConvBlock)     ← 完全一致
conv2  (ConvBlock)           conv2  (ConvBlock)     ← 完全一致
conv3  (ConvBlock)           conv3  (ConvBlock)     ← 完全一致
conv4  (ConvBlock)           conv4  (ConvBlock)     ← 完全一致
avgpool                      avgpool                ← 完全一致
classifier.1 (Linear 4096→256)  actor.0 (Linear 4096→256)  ← 对应
classifier.4 (Linear 256→6)     actor.2 (Linear 256→6)     ← 对应
(无)                         critic.0 (Linear 4096→256)  ← 新增
(无)                         critic.2 (Linear 256→1)     ← 新增
```

### 2.3 权重初始化区别

```python
# ActorCriticCNN 使用 orthogonal 初始化（适合 RL 策略网络）
def _init_weights(self):
    for m in self.modules():
        if isinstance(m, nn.Conv2d):
            nn.init.kaiming_normal_(m.weight, mode="fan_out")
        elif isinstance(m, nn.Linear):
            nn.init.orthogonal_(m.weight, gain=0.01)  # ← 不同于 SimpleCNN 的 normal_
```

**注意**：`_init_weights()` 在 `__init__` 最后调用。之后 `load_pretrained_backbone()` 会覆盖这些初始化值。所以预训练权重的初始化方式才是实际生效的。

## 3. load_pretrained_backbone 权重映射

### 3.1 映射表

```python
# 卷积骨干：同名直接映射
key_map = {
    "conv1.conv.weight": "conv1.conv.weight",
    "conv1.conv.bias":   "conv1.conv.bias",
    "conv2.conv.weight": "conv2.conv.weight",
    "conv2.conv.bias":   "conv2.conv.bias",
    "conv3.conv.weight": "conv3.conv.weight",
    "conv3.conv.bias":   "conv3.conv.bias",
    "conv4.conv.weight": "conv4.conv.weight",
    "conv4.conv.bias":   "conv4.conv.bias",
}

# Actor 头：从 classifier 映射
actor_head_map = {
    "actor.0.weight": "classifier.1.weight",  # Linear 4096→256
    "actor.0.bias":   "classifier.1.bias",
    "actor.2.weight": "classifier.4.weight",  # Linear 256→n_actions
    "actor.2.bias":   "classifier.4.bias",
}
```

### 3.2 跨层名称映射的原理

`nn.Sequential` 内部子模块按 0, 1, 2, ... 索引：
- `classifier[0]` = Flatten → state_dict 中为 `classifier.0.*`（无参数）
- `classifier[1]` = Linear(4096, 256) → `classifier.1.weight`, `classifier.1.bias`
- `classifier[4]` = Linear(256, 6) → `classifier.4.weight`, `classifier.4.bias`

Actor head 的 `nn.Sequential`：
- `actor[0]` = Linear(4096, 256) → `actor.0.weight`, `actor.0.bias`
- `actor[2]` = Linear(256, 6) → `actor.2.weight`, `actor.2.bias`

因此 `actor.0 ← classifier.1`, `actor.2 ← classifier.4`。

### 3.3 输出维度不匹配的处理

如果预训练模型输出 5 类（旧版无 long_press）而 ActorCriticCNN 输出 6 类：

```python
if ac_param.shape != pt_param.shape:
    if ".2." in ac_name and ac_param.dim() == 2:
        # actor.2.weight: (6, 256) vs classifier.4.weight: (5, 256)
        out_old = min(pt_param.shape[0], ac_param.shape[0])  # min(5, 6) = 5
        ac_param[:out_old] = pt_param[:out_old]   # 前 5 行复制
        # 第 6 行保持 _init_weights 的随机初始化
```

### 3.4 加载顺序

```python
# train_ppo.py train() 中的加载顺序
1. agent = PPOAgent()
   └→ ActorCriticCNN.__init__() → _init_weights() (orthogonal init)
2. agent.load_pretrained()
   └→ load_pretrained_backbone(network, pretrain_path)
      ├→ 覆盖 conv1-4 的 8 个参数 (kaiming_normal from pretrained)
      ├→ 覆盖 actor.0 的 2 个参数 (normal from pretrained classifier)
      └→ 覆盖 actor.2 的 2 个参数 (normal from pretrained classifier)
   └→ critic 头保持 orthogonal 初始化
```
