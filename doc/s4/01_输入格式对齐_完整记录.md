# S4-01 PPO 输入格式对齐 —— 完整技术记录

## 1. 问题发现

### 1.1 初始现象

用户运行 `python play_ai.py --pretrain`（纯 CNN 行为克隆模型）：
- 模型能**稳定躲避 5 个以上障碍物**
- 存活时间约 **1 分 30 秒**
- 说明预训练 CNN（SimpleCNN）质量很好

用户运行 `python train_ppo.py` 训练 PPO：
- 训练 40 分钟后，agent **只会输出 noop（不动）**
- 偶尔输出其他动作，但"瞎猫碰上死耗子"过一个障碍物就死
- 平均每局存活约 35 步（含 17 步暖机），真实游戏仅约 18 步

### 1.2 训练日志（第 27-29 局）

```
第 27 局 — 存活 35 步 | 奖励: -11.00
Step 18-37: 全部是 动作=不动 奖励=+0.00（20步连续 noop）
Step 38: 动作=不动 奖励=-1.00（红心丢失）
Step 39: 动作=no_op 奖励=-10.00 ★ 死亡

第 28 局 — 存活 39 步 | 奖励: -11.00
Step 18-30: 全部是 动作=不动（13步连续 noop）
Step 31: 红心丢失 → 死亡

第 29 局 — 存活 32 步 | 奖励: -11.00
Step 18-25: 动作=不动（8步）
Step 26-28: 动作=左移 奖励=+0.10（3步左移）
Step 29-34: 动作=不动（6步）
Step 35: 红心丢失 → 死亡
```

**关键观察**：agent 宁愿连续 20 步 noop 然后死掉，也不尝试做动作。说明奖励信号无法驱动 exploration。

### 1.3 教师模型异常

在 PPO 训练中，教师模型（用于提供蒸馏奖励）的行为：
```
Step 6: 动作=右移 → 教师说"左移"，扣 -0.5
Step 7: 动作=右移 → 教师说"左移"，扣 -0.5
Step 22: 动作=滑铲 → 教师说"左移"，扣 -0.5
Step 27: 动作=滑铲 → 教师说"左移"，扣 -0.5
Step 44: 动作=滑铲 → 教师说"跳跃"，扣 -0.5
```

教师几乎总预测同一类（主要是"左移"），偶尔变一下。但用户已验证同一模型在 `play_ai.py --pretrain` 中能正常玩——说明**不是模型坏了，而是 PPO 环境中的预处理与 play_ai 不一致**。

## 2. 根因分析

### 2.1 输入格式完全不一致

预训练 SimpleCNN 和 PPO ActorCriticCNN 的输入规格完全不同：

| 属性 | SimpleCNN（预训练） | ActorCriticCNN（PPO 旧） |
|---|---|---|
| 输入通道数 | **3**（单帧 RGB） | **15**（4帧×3通道 + 3跑道通道） |
| 输入分辨率 | **128×128** | **320×320**（FRAME_WIDTH/HEIGHT=320） |
| 帧数 | 1 帧 | 4 帧堆叠 |
| 附加信息 | 无 | 3 通道跑道 one-hot 编码 |
| 第一层卷积 | `Conv2d(3, 32, 3)` | `Conv2d(15, 32, 3)` |
| 参数量 | 约 4.3M | 约 4.4M |

### 2.2 权重加载是假象

`load_pretrained_backbone` 函数（旧版）的"加载"逻辑：

```python
# 旧版：conv1 形状不匹配时的部分复制
if ac_param.shape != pt_param.shape:
    if ac_name == "conv1.conv.weight":
        # (32, 15, 3, 3) vs (32, 3, 3, 3)
        ac_param[:, :3] = pt_param    # 只复制前 3/15 通道！
        # 其余 12 个通道保持随机初始化
```

这意味着：
- **conv1 层**：前 3 通道有预训练权重，后 12 通道完全随机
- **后面所有层**：虽然形状匹配，但输入来自一个半随机半预训练的 conv1，输出已经严重失真
- **Actor 头**：从预训练 classifier 复制了权重，但接收到的特征已经面目全非
- **Critic 头**：完全随机初始化

本质上，整个 PPO 网络**约等于随机初始化**。预训练权重的正向迁移**几乎为零**。

### 2.3 为什么教师模型也不准

教师模型是独立的 SimpleCNN，接收的输入是 3ch×128×128 的裁剪游戏画面。它的预处理在 `_teacher_predict` 中：

```python
# 旧版 _teacher_predict
cropped = _crop_game_area(raw_frame)    # 裁剪游戏区域
img = cv2.resize(cropped, (128, 128))   # 缩放到 128×128
# → 这个预处理与 play_ai.py CNN 路径完全相同
# → 但 agent 用的是 320×320 15通道输入
# → 教师预测 = argmax(SimpleCNN(128×128))
# → agent 预测 = sample(ActorCriticCNN(15ch×320×320))
# → 两个网络的"世界"完全不同 → 教师的预测对 agent 毫无参考价值
```

教师的预测在它自己的视角下是正确的，但 agent 的输入空间完全不同，agent 无法从教师预测中学到任何东西。

## 3. 修复方案

### 3.1 核心思路

**将 PPO 的 ActorCriticCNN 输入完全统一到预训练 SimpleCNN 的格式**：
- 3 通道（单帧 RGB）
- 128×128 分辨率
- 值域 [0, 1]（float32）
- 不做帧堆叠
- 不加入跑道编码

这样：
1. `load_pretrained_backbone` 中所有 8 层卷积权重 **100% 形状匹配**
2. Actor 头权重从 classifier **完整复制**
3. agent 初始化后的策略 ≈ 预训练模型的行为（能躲 5+ 障碍物）
4. 教师模型的预测对 agent 有直接参考价值

### 3.2 config.py 改动

**改动前**：
```python
FRAME_WIDTH = 320        # PPO 状态分辨率
FRAME_HEIGHT = 320
FRAME_STACK = 4          # 4 帧堆叠
# 第 81 行还有一个重复的 FRAME_STACK = 4  ← Bug，后面的覆盖前面的
```

**改动后**：
```python
FRAME_WIDTH = 128        # 与 IMG_WIDTH 一致
FRAME_HEIGHT = 128       # 与 IMG_HEIGHT 一致
FRAME_STACK = 1          # 单帧，与预训练一致
# 删除了重复的 FRAME_STACK = 4 定义
```

### 3.3 model.py 改动

**ActorCriticCNN 默认值**：
```python
# 改动前
class ActorCriticCNN(nn.Module):
    def __init__(self, input_channels=15, ...):  # 15 通道默认

# 改动后
class ActorCriticCNN(nn.Module):
    def __init__(self, input_channels=3, ...):   # 3 通道默认
```

**load_pretrained_backbone 简化**：
```python
# 旧版：处理 conv1 形状不匹配的分支逻辑
if ac_param.shape != pt_param.shape:
    if ac_name == "conv1.conv.weight":
        ac_param[:, :3] = pt_param  # 部分复制
# ↓ 改为 ↓
# 新版：直接检查形状，不匹配时报错而非静默部分复制
if ac_param.shape == pt_param.shape:
    ac_param.copy_(pt_param)
    loaded += 1
else:
    print(f"[Model] 形状不匹配 {ac_name}: {ac_param.shape} vs {pt_param.shape}")
```

### 3.4 train_ppo.py RLGameEnv 重写

**帧管理**——从列表变为单帧：

```python
# 改动前：帧堆叠列表
self._frame_stack = []
def _init_frame_stack(self):
    for _ in range(config.FRAME_STACK):  # 4 次
        raw = get_screen_numpy()
        processed = preprocess_frame(raw)  # → (3, 320, 320)
        self._frame_stack.append(processed.copy())
def _push_frame(self, processed):
    self._frame_stack.pop(0)
    self._frame_stack.append(processed)

# 改动后：单帧
self._current_frame = None  # (3, 128, 128)
def _init_frame(self):
    raw = get_screen_numpy()
    cropped = _crop_game_area(raw)
    resized = cv2.resize(cropped, (128, 128))
    self._current_frame = resized.astype(np.float32).transpose(2,0,1) / 255.0
```

**get_state 重写**：

```python
# 改动前：(3*4+3, 320, 320) = (15, 320, 320)
def get_state(self):
    state = np.concatenate(self._frame_stack, axis=0)  # (12, 320, 320)
    lane = np.zeros((3, 320, 320))
    lane[self._current_lane - 1] = 1.0
    state = np.concatenate([state, lane], axis=0)      # (15, 320, 320)
    return state

# 改动后：(3, 128, 128)
def get_state(self):
    return self._current_frame.copy()
```

### 3.5 play_ai.py PPO 预测路径改动


## 4. 验证

改动后的预期行为：
1. `load_pretrained_backbone` 应打印 `骨干: 8/8 层匹配`（全部成功）
2. Agent 初始策略应与预训练模型一致（能躲 5+ 障碍物）
3. 教师模型预测与 agent 预测应有高度一致性（>80%）
4. PPO 训练应在已有良好策略基础上微调，而非从零探索
