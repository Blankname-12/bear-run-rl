# S4-05 探索度不足、熵系数与策略坍缩 —— 完整记录

## 1. 问题现象

PPO 训练若干局后，agent 陷入固定的行为模式：**"左左停右右停"循环往复**。

```
Step 18-19: 左移 ×2
Step 20:    不动
Step 21-22: 右移 ×2
Step 23:    不动
Step 24-25: 左移 ×2
Step 26:    不动
...循环...
```

同时，agent **几乎从不做滑铲**（action 1, swipe_down）。

## 2. 根因分析

### 2.1 策略坍缩的机制

PPO 的策略更新方向是最大化优势函数的期望。当策略收敛到一组能获得正奖励的动作序列后，梯度会继续加强这些动作，导致策略越来越"尖锐"（softmax 分布越来越集中在少数动作上）。

理论上，PPO 的 entropy bonus 应该阻止这种坍缩：

```python
total_loss = actor_loss + critic_coef * critic_loss + ENTROPY_COEF * entropy_loss
# entropy_loss = -entropy.mean()
# 负号 = 最大化熵 = 鼓励均匀分布
```

但 `ENTROPY_COEF = 0.02` 太小，entropy bonus 不足以对抗 actor loss 的坍缩压力。

### 2.2 为什么滑铲不出

可能原因（按优先级排序）：
1. **预训练模型对滑铲识别率低**：标注数据中滑铲样本少 → 预训练 CNN 很少预测滑铲 → 教师也很少预测滑铲 → agent 探索到滑铲也无法匹配教师 → 拿不到 +1.0 奖励
2. **滑铲的视觉特征不显著**：在单帧图像中，滑铲和 noop 可能看起来相似（都是角色在跑道中间），区别在于动态 motion，但 PPO 使用的是**单帧**输入
3. **熵太小 → 不会主动探索少见动作**

### 2.3 为什么形成"左左停右右停"定式

1. 左移/右移的视觉特征最明显（画面明显偏向一侧）
2. 预训练模型学会了区分左移和右移
3. Agent 做左移/右移 → 大概率匹配教师 → 得 +1.0
4. 连续做同向动作 → 第 2 次被防抖跳过 → 第 3 次执行 → 形成"两次同向+一次停"的节奏
5. PPO 强化这个模式 → 策略坍缩

## 3. 修复：大幅提高熵系数

### 3.1 改动

```python
# config.py
# 改动前
ENTROPY_COEF = 0.02

# 改动后
ENTROPY_COEF = 0.15    # 7.5x 提升
```

### 3.2 效果预期

更大的 ENTROPY_COEF 意味着：
1. PPO 损失函数中熵项的权重更大 → 梯度更强地推动策略向均匀分布
2. Agent 更愿意尝试之前很少做的动作（如滑铲）
3. 策略不会太"尖锐"，维持一定的随机性
4. 打破"左左停右右停"的定式循环

### 3.3 熵系数的作用原理

```python
# PPO 中的 entropy 计算
probs = F.softmax(logits, dim=-1)           # [0.1, 0.05, 0.3, 0.3, 0.2, 0.05]
entropy = -(probs * log(probs)).sum(dim=-1) # 熵值

# 当策略坍缩时 (如 [0.9, 0, 0, 0, 0.1, 0])
# entropy ≈ -(0.9*log(0.9) + 0.1*log(0.1)) ≈ 0.33
# entropy_loss = -0.33（加到 total_loss → 惩罚低熵）

# 当策略均匀时 (如 [1/6, ..., 1/6])
# entropy ≈ -6*(1/6*log(1/6)) ≈ 1.79
# entropy_loss = -1.79（较大的负值 → 奖励高熵）

# ENTROPY_COEF = 0.15 时
# 坍缩策略的 entropy 惩罚: -0.15 * 0.33 = -0.05（小惩罚）
# 均匀策略的 entropy 奖励: -0.15 * 1.79 = -0.27（大奖励）
# 差异: 0.22，足以影响梯度方向
```

### 3.4 潜在风险与对策

**风险**：熵太大 → 策略过于随机 → 表现下降（甚至不如预训练基线）

**对策**：
- 如果出现此情况，可在训练中后期逐步衰减 ENTROPY_COEF
- 例如每 50 局衰减 0.01，从 0.15 → 0.05
- 衰减逻辑可在 train() 主循环中加入

```python
# 可选：熵衰减（未实施，如有需要可加）
if agent.episodes % 50 == 0:
    config.ENTROPY_COEF = max(0.03, config.ENTROPY_COEF - 0.01)
```

## 4. 滑铲问题的补充分析

即使熵提高了，如果预训练模型本身就很少预测滑铲，那么：
- PPO 的 entropy bonus 会让 agent 更多尝试滑铲
- 但尝试滑铲时 → 教师预测不是滑铲 → 不匹配 → 得 0 分（不扣不奖）
- 如果滑铲碰巧导致更好的结果（存活更长），死亡惩罚减少 → 自然强化
- 如果滑铲导致更差的结果（立刻死），死亡惩罚 → 自然削弱

**结论**：entropy 是必要的，但滑铲能否出现还取决于：
1. 教师是否认为滑铲是好的（预训练模型的选择）
2. 滑铲在实际游戏中是否真的有用（环境的反馈）

如果滑铲确实是这个游戏中正确的应对策略，但预训练模型没学到，那么只有两种办法：
- 重新采集更多滑铲标注数据
- 或等 PPO 通过 trial-and-error 发现（需要大量探索 + 足够的 entropy）
