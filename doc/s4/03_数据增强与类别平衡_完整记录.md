## 2. 数据增强演进

### 2.1 初始方案（原始代码）

```python
def _augment(self, img, action):
    # 1. 水平翻转 (p=0.5)：left(2) ↔ right(3)
    if random.random() < 0.5:
        img = cv2.flip(img, 1)
        if action == 2: action = 3
        elif action == 3: action = 2

    return img, action
```

仅 2 种增强，使用 cv2 操作 numpy 数组。

### 2.3 用户反馈："水平/垂直翻转都是没必要的"

**用户的核心论点**：游戏画面永远不会水平翻转或垂直翻转。Temple Run 类游戏：
- 摄像机固定在角色后方
- 视角永远是正的（无镜像、无旋转）
- 水平翻转后的画面（树在左边变成在右边、文字反转）**在真实游戏中永远不会出现**
- 在翻转图像上学到的特征 → 部署到真实游戏 → 特征不存在 → 误导模型

### 2.4 收窄为仅颜色/模糊增强

**最终方案**（当前代码）：

```python
def _augment(self, img, action):
    # 仅做颜色/模糊增强，不做空间变换
    import PIL.ImageEnhance, PIL.ImageFilter

    # 1. 亮度抖动 (p=0.5, factor 0.6-1.4)
    #    理由：模拟屏幕亮度变化、色温调整
    if random.random() < 0.5:
        factor = random.uniform(0.6, 1.4)
        img = PIL.ImageEnhance.Brightness(img).enhance(factor)

    # 2. 对比度抖动 (p=0.3, factor 0.7-1.3)
    #    理由：不同模拟器的色彩配置差异
    if random.random() < 0.3:
        factor = random.uniform(0.7, 1.3)
        img = PIL.ImageEnhance.Contrast(img).enhance(factor)

    # 3. 饱和度抖动 (p=0.3, factor 0.5-1.5)
    #    理由：画面色彩偏差
    if random.random() < 0.3:
        factor = random.uniform(0.5, 1.5)
        img = PIL.ImageEnhance.Color(img).enhance(factor)

    # 4. 高斯模糊 (p=0.15, radius 0.3-0.8)
    #    理由：模拟运动模糊、低质量屏幕的压缩噪声
    if random.random() < 0.15:
        img = img.filter(PIL.ImageFilter.GaussianBlur(
            radius=random.uniform(0.3, 0.8)))

    return img, action
```

### 2.5 被移除的增强及其理由

| 被移除 | 移除理由 |
|---|---|
| 水平翻转 | 游戏画面不会镜像，left↔right 标签对调学到的特征不存在于真实游戏 |
| 随机裁剪+缩放 | 游戏 UI 布局固定，裁剪可能切掉障碍物或角色 UI，造成标签与图像不一致 |
| 锐化 | 效果不可控，容易引入 JPEG 伪影的过增强 |

## 3. 类别平衡策略

### 3.1 问题

标注数据中 noop（不动）占绝对多数（通常 50-70%）。如果不处理，模型会学到"总是预测 noop"的懒惰策略，因为这样就能拿到 50-70% 的准确率。

### 3.2 noop 降采样

```python
def _balance(self):
    """noop (action=4) 降采样到其他类别中位数"""
    by_action = {a: [] for a in range(6)}
    for img_path, action in self.samples:
        by_action[action].append((img_path, action))

    # 取非 noop 类别样本数的中位数
    non_noop = [0, 1, 2, 3, 5]  # 跳,滑,左,右,长按
    other_counts = [len(by_action[a]) for a in non_noop]
    max_non_noop = int(np.median(other_counts))

    # noop 数量 > 中位数 → 随机降采样
    if max_non_noop > 0 and len(by_action[4]) > max_non_noop:
        kept = random.sample(by_action[4], max_non_noop)
        balanced = []
        for a in non_noop:
            balanced.extend(by_action[a])
        balanced.extend(kept)  # 降采样后的 noop
        self.samples = balanced
```

### 3.3 类别加权损失（新增）

降采样只能让 noop 和其他类数量相近，但不能处理其他类之间的不平衡（如滑铲可能比跳跃少）。需要额外的类别加权：

```python
# 在 pretrain() 训练函数中，计算类别权重
train_counts = Counter(train_full.samples[i][1] for i in train_idx.indices)
max_count = max(train_counts.values())
class_weights = torch.zeros(6)
for a in range(6):
    class_weights[a] = max_count / max(train_counts.get(a, 1), 1)
class_weights = class_weights.to(device)

# 使用加权交叉熵
criterion = nn.CrossEntropyLoss(weight=class_weights)
```

**效果**：样本数少的类别（滑铲、长按）获得更大的损失权重，模型被迫更认真地学习这些类别。

权重计算例：
```
跳跃: 5000 张 → weight = 5000/5000 = 1.0
滑铲: 1000 张 → weight = 5000/1000 = 5.0  ← 5x 权重
左移: 3000 张 → weight = 5000/3000 = 1.67
长按: 500 张  → weight = 5000/500  = 10.0 ← 10x 权重
```

## 4. 其他技术改进

### 4.1 PIL 全流程

将所有图像 I/O 从 cv2 改为 PIL：
- **原因**：cv2.imread/imwrite **不支持中文路径**（Windows 下路径含中文字符会返回 None）
- **改动**：`Image.open()` 代替 `cv2.imread()`，`img.resize()` 代替 `cv2.resize()`
- **注意事项**：PIL 读入是 RGB，cv2 读入是 BGR，需要统一
