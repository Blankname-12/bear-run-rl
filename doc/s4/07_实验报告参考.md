## 关键技术决策及理由

1. **统一输入为 3ch×128×128**：预训练权重 100% 复用，否则 PPO 等于从零开始
2. **教师只给糖不给鞭子**（MISMATCH=0）：探索不应被惩罚
3. **noop 小罚 -0.1**：足够形成梯度方向，不会扼杀探索
4. **死亡 -1.0 而非 -10.0**：轻罚避免 agent 因恐惧而完全不动作
5. **去掉红心丢失扣分**：检测有滞后，扣分无意义
6. **ENTROPY_COEF=0.15**：策略坍缩到定式循环后的大幅提升
7. **防抖 skip 2nd / allow 3rd**：防双击但不阻止持续操作
8. **长按用 subprocess.run**：同步等待，防止重叠触摸事件
9. **数据增强去掉空间变换**：游戏画面不会翻转/旋转/裁剪
10. **类别加权 + noop 降采样**：双重保护防止类别不平衡
