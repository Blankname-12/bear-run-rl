# 01 — 项目概述

## 项目背景

使用强化学习（PPO）+ 行为克隆（BC）训练智能体自动游玩 Temple Run 类跑酷游戏"熊大快跑"。

**核心挑战**：游戏运行在 Android 模拟器上，所有交互（截屏、触控）必须通过 ADB 中转，实时性要求高。

## 技术路线

```
手动数据采集 ──→ 行为克隆预训练 ──→ PPO 强化学习微调
collect_data.py   pretrain.py        train.py
```

1. **数据采集**：人类玩家键盘（W/A/S/D）操控游戏，记录（状态, 动作）对
2. **行为克隆**：监督学习训练 CNN 模仿人类操作（CrossEntropyLoss）
3. **PPO 微调**：预训练权重基础上自主探索、优化策略

## 系统架构

```
┌──────────────────────────────────────────┐
│              Python 控制端（Windows）      │
│                                           │
│  FastTouch ──→ adb exec-out ──→ 触控      │
│  Minicap   ←── TCP socket ←── ADB Forward│
│                                           │
│  GameEnv   ── 游戏环境（step/observe）     │
│  PPOAgent  ── PPO Actor-Critic           │
└──────────────┬───────────────────────────┘
               │ ADB Server (tcp:5037)
┌──────────────┴───────────────────────────┐
│      雷电模拟器 (127.0.0.1:5555)         │
│      Android 9.0  x86_64  SDK 28        │
│      1080×1920 原生竖屏                  │
│                                           │
│  minicap 服务端 ─ 15fps JPEG 流           │
│  input swipe   ── Android 触控系统        │
│  「熊大快跑」游戏                          │
└──────────────────────────────────────────┘
```

## 性能指标

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 截屏延迟 | 682ms (ADB screencap) | ~17ms (minicap 15fps) |
| 触控延迟 | 182ms (subprocess.run shell) | ~5ms (Popen exec-out) |
| 图像处理 | ~50ms (PIL) | ~10ms (OpenCV) |
| 端到端帧率 | ~1 FPS | ~20 FPS |
| 截屏分辨率 | 窗口缩放 | 原生 1080×1920 |
| 动作空间 | 4 (无no-op) | 5 (含no-op) |
| 死亡检测 | 单帧HSV | 3帧投票+二次验证 |
