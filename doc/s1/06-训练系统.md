# 06 — 训练系统

## 整体流程

```
采集数据 → 行为克隆预训练 → PPO 强化学习微调
```

**为什么需要预训练**：
- PPO 从头训练在复杂游戏环境中探索效率极低
- 随机初始化的 CNN 完全随机操作，角色几秒就死，永远收集不到有效的长期奖励信号
- 行为克隆先让 CNN 学会基本的"看到障碍物→做出反应"模式，PPO 在此基础上优化

### 训练过程

```python
def pretrain():
    states = np.load(states_path)   # (N, 15, 128, 128)
    actions = np.load(actions_path) # (N,)

    model = PPONetwork(input_channels=15, n_actions=5)
    dataset = TensorDataset(torch.FloatTensor(states), torch.LongTensor(actions))
    loader = DataLoader(dataset, batch_size=128, shuffle=True)

    criterion = nn.CrossEntropyLoss()
    optimizer = Adam(model.parameters(), lr=1e-3)
    scheduler = CosineAnnealingLR(optimizer, epochs)

    for epoch in range(epochs):
        for batch_states, batch_actions in loader:
            logits, _ = model(batch_states)
            loss = criterion(logits, batch_actions)
            loss.backward()
            optimizer.step()
        scheduler.step()
        acc = (logits.argmax(1) == batch_actions).float().mean()
```

### 超参

| 参数 | 值 | 依据 |
|------|-----|------|
| Epochs | 30 | 约 200 条数据下，30 轮基本收敛 |
| Batch Size | 128 | 数据量小时全量拟合 |
| Learning Rate | 1e-3 | 监督学习标准值 |
| Scheduler | CosineAnnealing | 平滑降低 LR，避免震荡 |
| Loss | CrossEntropy | 多分类标准损失 |

## PPO 训练（train.py + agent.py）

### 模型架构

Nature CNN（与 DQN 的 Nature 论文相同结构，改编为 Actor-Critic 双头）：

```python
class PPONetwork(nn.Module):
    def __init__(self, input_channels=15, n_actions=5):
        super().__init__()
        self.conv = nn.Sequential(
            nn.Conv2d(input_channels, 32, kernel_size=8, stride=4),  # 128→31
            nn.ReLU(),
            nn.Conv2d(32, 64, kernel_size=4, stride=2),              # 31→14
            nn.ReLU(),
            nn.Conv2d(64, 64, kernel_size=3, stride=1),              # 14→12
            nn.ReLU(),
        )
        conv_out = 12 * 12 * 64  # 9216

        self.actor = nn.Sequential(
            nn.Linear(conv_out, 256), nn.ReLU(),
            nn.Linear(256, n_actions),  # 输出 5 维 logits
        )
        self.critic = nn.Sequential(
            nn.Linear(conv_out, 256), nn.ReLU(),
            nn.Linear(256, 1),          # 输出标量 V(s)
        )
```

**权重初始化**：actor 和 critic 的全连接层使用正交初始化（orthogonal initialization），gain=√2，bias=0。正交初始化在强化学习中比 Kaiming/Xavier 更稳定（OpenAI Baselines 的做法）。

### RolloutBuffer

```python
class RolloutBuffer:
    def push(self, state, action, log_prob, reward, value, done):
        self.states.append(state)
        self.actions.append(action)
        self.log_probs.append(log_prob)
        self.rewards.append(reward)
        self.values.append(value)
        self.dones.append(done)

    def get_all(self):
        return (np.array(self.states), np.array(self.actions),
                np.array(self.log_probs), np.array(self.rewards),
                np.array(self.values), np.array(self.dones))
```

### GAE 优势估计

```python
def _compute_gae(self, rewards, values, dones, last_value):
    advantages = np.zeros(len(rewards), dtype=np.float32)
    gae = 0.0
    for t in reversed(range(len(rewards))):
        next_value = last_value if t == len(rewards)-1 else values[t+1]
        delta = rewards[t] + GAMMA * next_value * (1 - dones[t]) - values[t]
        gae = delta + GAMMA * GAE_LAMBDA * (1 - dones[t]) * gae
        advantages[t] = gae
    returns = advantages + values
    return advantages, returns
```

GAE（Generalized Advantage Estimation）用指数衰减的方式平衡偏差和方差。λ=0.95 意味着约 20 步的信用分配窗口。

### PPO Clip 损失

```python
# Actor loss (PPO clipped objective)
ratio = exp(new_log_probs - old_log_probs)
surr1 = ratio * advantages          # 无约束
surr2 = clamp(ratio, 0.8, 1.2) * advantages  # 裁剪
actor_loss = -min(surr1, surr2).mean()

# Critic loss
critic_loss = F.mse_loss(values, returns)

# Entropy bonus (鼓励探索)
entropy_loss = -entropy.mean()

# Total loss
loss = actor_loss + 0.5 * critic_loss + 0.02 * entropy_loss
```

裁剪系数 ε=0.2 意味着新旧策略的概率比限制在 [0.8, 1.2]，防止单次更新步幅过大。

### PPO 超参

| 参数 | 值 | 依据 |
|------|-----|------|
| LEARNING_RATE | 3e-4 | PPO 标准 LR |
| GAMMA | 0.99 | 长视距任务（需要看到远处的障碍物） |
| GAE_LAMBDA | 0.95 | 平衡偏差和方差 |
| CLIP_EPSILON | 0.2 | OpenAI 默认值 |
| ENTROPY_COEF | 0.02 | 略高（鼓励探索，避免过早收敛到次优） |
| CRITIC_COEF | 0.5 | 标准值 |
| MAX_GRAD_NORM | 0.5 | 梯度裁剪防爆炸 |
| ROLLOUT_STEPS | 512 | 单局约 40-50 步，512 步约 10 局 |
| PPO_EPOCHS | 4 | 标准值 |
| MINI_BATCH | 64 | 256/512 样本分 4/8 个 mini-batch |

### 奖励函数

```python
REWARD_ALIVE = +0.5      # 每步存活
REWARD_DEATH = -10.0     # 死亡

# 障碍物逼近检测（帧差法）
# 在屏幕下方的"危险区"检测像素变化
DANGER_ZONE_X1 = 0.3     # 左 30%
DANGER_ZONE_X2 = 0.7     # 右 70%
DANGER_ZONE_Y1 = 0.78    # 上 78%（障碍物进入就很近）
DANGER_ZONE_Y2 = 0.95    # 下 95%

def _check_danger(self, raw_frame):
    cropped = _crop_game_area(raw_frame)
    curr_gray = _extract_danger_zone_gray(cropped)
    if self._prev_danger_gray is not None:
        diff = abs(curr_gray - self._prev_danger_gray).mean()
        if diff > DANGER_FRAME_DIFF_THRESH:  # 8.0
            return REWARD_DANGER  # -0.3
    return 0.0
```

**帧差法**：对比连续两帧在"危险区"的像素变化。如果变化剧烈，说明有新物体（障碍物）进入了危险区。这给智能体一个"有东西接近了"的负奖励信号。

**无位移惩罚**：当智能体已经在最左/最右赛道，再次向左/右滑会无位移。这种情况下奖励为 0（不是惩罚，但不给生存奖励）。

### 训练循环

```python
while not interrupted:
    state = env.reset()
    episode_reward = 0.0
    done = False

    while not done and not interrupted:
        action, log_prob, entropy, value = agent.select_action(state)
        next_state, reward, done = env.step(action)
        agent.store_transition(state, action, log_prob, reward, value, done)
        state = next_state
        episode_reward += reward
        agent.update()  # 攒够 512 步触发 PPO 更新

    # 局结束
    episode_rewards.append(episode_reward)
    avg100 = mean(episode_rewards[-100:])
    avg20 = mean(episode_rewards[-20:])

    # 最佳模型
    if avg20 > best_avg_reward:
        agent.save_best_model()

    # 定期保存
    if episode % 30 == 0:
        agent.save_temp_model(f"ep{episode}")
        agent.save_checkpoint()

    # 死亡后重启
    env.restart_game()
```
