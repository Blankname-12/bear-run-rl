# 奖励设计四轮迭代

## 第一轮: 真机 + 帧差法（已废弃）
```python
REWARD_ALIVE = 0.2       # 存活
REWARD_DEATH = -10.0     # 死亡
REWARD_AVOID = 1.0       # 正确闪避(帧差法检测)
REWARD_MISS = -2.0       # 障碍物逼近不闪避
REWARD_CLEAR = 0.05      # 前方无障碍
```

### 致命bug
帧差法在 STEP_INTERVAL=500ms 时两帧场景变化巨大, 检测全为噪声。
`obstacle_in_my_lane and action != no_op → +1.0` 让智能体学到"随便滑就能拿分"。

## 第二轮: 四个干净信号
```python
REWARD_ALIVE = 0.2       # 存活
REWARD_DEATH = -10.0     # 死亡
REWARD_HEART_LOST = -0.5 # 红心丢失(每帧累加 -0.5×帧数)
REWARD_BOUNDARY = -0.5   # 边界无效滑动
```
移除帧差法和所有障碍物相关代码。
问题: 纯生存信号, 智能体学会"不动最安全", 活~115步到顶。

## 第三轮: 参数爆炸（exp40 惨案）
```python
REWARD_HEART_LOST = -10   # 每帧 -10！2帧 = -20
HEART_DEATH_FRAMES = 2    # 太快确认死亡
STEP_INTERVAL = 1.2       # 反应窗口太窄
```
结果: 855局, 平均19步(22秒), 99.9%暴毙。
智能体收敛到"早死早超生" — 死快 ≈ -18奖励, 活久反而扣更多。

## 第四轮: 仿真环境最终方案
```python
reward = 0.0
reward += score_delta * 0.1      # 分数增量(主要信号, 每步可算)
reward += 0.01                    # 存活奖励
reward -= 0.3                     # 撞墙(边界无效滑动)
reward += 0.5                     # 闪避成功(障碍物离开屏幕)
reward += 1.0                     # 收集金币
reward -= 5.0                     # 碰撞死亡
```
为何有效: 分数增量/闪避成功/金币收集提供密集即时正反馈, 死亡惩罚适中不压倒正信号, 完美信息让信用分配精确。

## 关键教训
1. **HEART_LOST 惩罚不能太重**: 它叠加在 death 前几帧, 太大会导致"早死早超生"
2. **STEP_INTERVAL 不能太长**: 障碍物从出现到碰撞 ~1-2秒, 间隔太长反应窗口不足
3. **纯生存奖励不够**: 需要即时动作反馈 (分数/闪避/金币)
4. **帧差法在长间隔下完全不可用**: 必须走视觉CNN或内存直接读
