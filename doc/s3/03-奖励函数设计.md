# 奖励函数设计全记录

## 第一版: 帧差法 + PPO（真机）

### 配置
```python
REWARD_ALIVE = 0.2        # 存活 (每步)
REWARD_DEATH = -10.0      # 死亡
REWARD_AVOID = 1.0        # 正确闪避障碍物
REWARD_MISS = -2.0        # 障碍物逼近但不闪避
REWARD_CLEAR = 0.05       # 前方无障碍鼓励前进
```

### 障碍物检测: 帧差法
```python
def _detect_obstacles(prev_cropped, curr_cropped):
    # RGB → 帧差 → 灰度 → 二值化 → 形态学去噪 → 找轮廓
    diff = cv2.absdiff(prev_cropped, curr_cropped)
    gray = cv2.cvtColor(diff, cv2.COLOR_RGB2GRAY)
    _, binary = cv2.threshold(gray, 25, 255, cv2.THRESH_BINARY)
    kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
    cleaned = cv2.morphologyEx(binary, cv2.MORPH_OPEN, kernel)
    contours, _ = cv2.findContours(cleaned, cv2.RETR_EXTERNAL, ...)
    # 按赛道分类 → 返回 [(lane, y_ratio, area), ...]
```

### 帧差法奖励逻辑（有致命 bug）
```python
def _compute_obstacle_reward(self, raw_frame, action_idx):
    obstacles = _detect_obstacles(self._prev_cropped, cropped)
    if not obstacles:
        if action_idx == 4:  return REWARD_CLEAR     # +0.05
        else:                return -REWARD_CLEAR    # -0.05

    obstacle_in_my_lane = any(lane == my_lane for lane, _, _ in obstacles)
    if obstacle_in_my_lane:
        if action_idx == 4:  return REWARD_MISS      # -2.0
        else:                return REWARD_AVOID     # +1.0  ← BUG
    else:
        if action_idx == 4:  return REWARD_CLEAR     # +0.05
        else:                return -REWARD_CLEAR    # -0.05
```

### BUG 分析
帧差法在 500ms 间隔下检测到的全是噪声（场景变化太大），`obstacle_in_my_lane` 几乎总是 True。
→ "有障碍物" + "做了任意非 no_op 动作" = +1.0
→ 智能体学到"随便滑就能拿分"，而不是"精确闪避障碍物"

### 为何噪声主导学习
噪声信号振幅: ±2.0（AVOID/MISS），存活信号振幅: +0.2
→ 噪声是存活信号的 **10 倍**
→ PPO 无法分离有效信号和噪声

## 第二版: 四个干净信号（移除帧差法）

### 配置
```python
REWARD_ALIVE = 0.2       # 存活 (每步)
REWARD_DEATH = -10.0     # 死亡
REWARD_HEART_LOST = -0.5 # 红心丢失 (每帧累加)
REWARD_BOUNDARY = -0.5   # 边界无效滑动
```

### 奖励计算
```python
# step_reward (每步基础)
if 左车道左滑 or 右车道右滑:
    step_reward = -0.5    # 撞墙
else:
    step_reward = +0.2    # 存活

# 红心丢失额外惩罚
if self._heart_loss_frames > 0:
    reward += -0.5 * self._heart_loss_frames

# 死亡时整个 episode 奖励为 -10
if dead:
    return next_state, -10.0, True
```

### 为什么不够
- 存活 +0.2/步 vs 死亡 -10: 需要活 50 步才能抵消一次死亡
- **没有即时的障碍物闪避反馈**: 智能体不知道"这步的左滑救了命"
- GAE 回溯范围 ~17 步(8.5 秒): 死亡前 17 步内的动作才有梯度
- 预训练 no_op bias → PPO 探索暴毙 → 收敛到"不动最安全"

### 红心丢失惩罚设计
红心丢失 ≠ 死亡。每帧红心不在就给惩罚，形成梯度信号:
```
正常跑 → 红心丢1帧(-0.5) → 丢5帧(-2.5) → 丢10帧确认死亡(-5-10=-15)
```
比纯死亡 -10 有更多中间反馈。但信号仍是"被动的"（红心掉是因为碰撞，但碰撞和红心掉之间有几帧延迟）。

## 第三版: 参数爆炸（exp40）

### 配置
```python
REWARD_HEART_LOST = -10   # 从 -0.5 → -10，×20
STEP_INTERVAL = 1.2       # 从 0.5 → 1.2，动作频率减半
HEART_DEATH_FRAMES = 2    # 维持
```

### 为什么失败
死前两步的时间线:
```
步骤 17: 红心可见, 奖励 +0.2, _heart_loss_frames=0
步骤 18: 碰撞！红心消失, _heart_loss_frames=1, 奖励 = 0.2 + (-10×1) = -9.8
步骤 19: 红心仍消失, _heart_loss_frames=2, 死亡确认, 奖励 = -10.0
```
全 episode 奖励: +3.4(17步) + (-9.8) + (-10) = **-16.4**

智能体学到: "红心掉了 = 巨亏 -10/步 → 千万别让红心掉 → 但怎么防止？不知道 → 少动少出错 → no_op → 仍会撞 → 死循环"

reward 分布 80% 集中在 -18.6 到 -17.6，极窄 → 无学习梯度

## 第四版: 仿真环境（当前）

### 配置
```python
reward = 0.0

# 1. 分数增量 (主要信号, 每步可精确计算)
score_delta = current_score - prev_score
if score_delta > 0:
    reward += score_delta * 0.1

# 2. 存活奖励 (微弱, 避免智能体停滞)
reward += 0.01

# 3. 撞墙惩罚 (边界无效滑动)
if action in (2, 3) and lane == lane_before:
    reward -= 0.3

# 4. 闪避成功 (障碍物离开屏幕底部, 确认没撞到)
# 在障碍物管理中: wy < -80 → obstacles.pop() → reward += 0.5

# 5. 收集金币
if coin_collected:
    reward += 1.0

# 6. 死亡惩罚
if collision:
    reward -= 5.0
```

### 为什么有效

#### 即时反馈密度
仿真每步都有反馈:
- 闪避成功立即 +0.5（障碍物离开屏幕的步骤）
- 金币收集立即 +1.0
- 碰撞立即 -5.0（撞即死，无延迟）
- 分数增量 +0.1×speed 提供持续正反馈

真机的问题（碰撞→红心下降→死亡 有 2-5 秒延迟）被彻底消除。

#### 信用分配精确
障碍物从生成(wy=900)到碰撞(wy~500)约 400/speed 步。在仿真中，每个障碍物的生命周期完全可追踪。闪避成功知道是哪个动作避免了碰撞，碰撞知道是哪个动作没避开。

#### 奖励量级平衡
| 事件 | 奖励 | 等效存活步数 |
|------|------|------------|
| 存活 1 步 | +0.01 | 1 |
| 闪避成功 | +0.5 | 50 |
| 收集金币 | +1.0 | 100 |
| 撞墙 | -0.3 | -30 |
| 死亡 | -5.0 | -500 |

正反馈足够密集，负反馈不会压死探索。

### v1: 固定时间
```python
GAME_START_WARMUP_SECONDS = 9.0
# 开局前 9 秒只观测不操作
if time.time() < self._warmup_deadline:
    return next_state, 0.0, False
```

### v2: 红心检测（本会话实现）
```python
# 暖机结束 = 红心首次出现
if not self._heart_seen:
    heart_px = _detect_heart_pixels(raw)
    if heart_px / region_area >= HEART_PIXEL_RATIO:
        self._heart_seen = True  # 暖机结束, 正式开始
    else:
        return next_state, 0.0, False  # 继续等
```
好处: 不依赖固定时间（有时加载快有时慢），红心出现是客观的"游戏开始"信号。加载动画、死亡动画、广告弹窗期间红心不可见 → 自动跳过这些无效画面。
