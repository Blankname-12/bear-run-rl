# 技术踩坑全集

> 会话3 遇到并修复的所有问题

## 1. cv2.imwrite 中文路径静默失败
- 症状: `cv2.imwrite` 返回 True 但文件不存在
- 原因: OpenCV C API 不支持 Unicode 路径
- 修复: `cv2.imencode('.png', img)[1].tofile(path)`

## 2. np.frombuffer 只读数组
- 症状: `cv2.rectangle(frame, ...)` 报 "NumPy array marked as readonly"
- 原因: `np.frombuffer` 返回 bytes 的只读视图
- 修复: `.copy()` 或 `np.ascontiguousarray()`

## 3. numpy 切片后不连续
- 症状: "Layout of the output array img is incompatible with cv::Mat"
- 原因: 从 (h,w,4) 切片 [:,:,:3] 后 stride 不连续
- 修复: `np.ascontiguousarray(img[:,:,:3])`

## 4. pygame 中文字体崩溃
- 症状: `pygame.font.SysFont("consolas",18)` → UnicodeDecodeError in initsysfonts_win32
- 原因: pygame 扫描系统字体时, 中文用户名路径触发 GBK 编码错误
- 修复: `pygame.font.Font(None, 20)` 使用内置默认字体

## 5. 不可见字符 U+200B
- 症状: `SyntaxError: invalid non-printable character U+200B`
- 原因: Write 工具生成代码时混入零宽空格
- 修复: 删除重写

## 6. "windows" vs "window"
- 症状: 截屏感觉巨慢
- 原因: `CAPTURE_MODE = "windows"` 多写 s, 不匹配任何模式, 降级到 ADB 1FPS
- 修复: 改为 `"window"`

## 7. scrcpy header 解析乱码
- 症状: 设备名乱码, 分辨率 31330×51213
- 原因: `send_device_meta=false` 时不发元数据, 把 H.264 裸流当 header 解析
- 修复: 跳过 header 解析, 直接用 config.SCREEN_WIDTH/HEIGHT

## 8. minicap header 垃圾分辨率
- 症状: 70778880×125829120
- 原因: 同上, banner 格式与代码不匹配
- 修复: 跳过 header, 只用 PID 确认连接, 分辨率用 config 值

## 9. minicap get_frame 死等
- 症状: `_recv_exact` 无超时, 断连后永久阻塞
- 修复: 加 3秒超时 + 连续5次失败自动重建 + 每120秒主动重启

## 10. 心跳检测3帧截屏拖慢速度
- 症状: 每步 3 次 minicap 截屏 = 300ms+ 额外等待
- 修复: step() 调用 `_heart_lost(raw, fast=True)` 单帧检测

## 11. ADB text=True 崩溃
- 症状: subprocess 后台线程 UnicodeDecodeError in GBK
- 原因: adb push 输出二进制进度, 无法用 GBK 解码
- 修复: adb 命令用 `text=False` (bytes), 需要文本时手动 utf-8 decode

## 12. conda run 不支持多行参数
- 症状: `AssertionError: Support for scripts where arguments contain newlines not implemented`
- 修复: 改用独立 .py 脚本文件

## 13. 旧模型加载到新网络报错
- 症状: `Missing key(s): "conv.0.weight"... Unexpected key(s): "net.0.weight"...`
- 原因: play.py 用 `--visual` 默认 CNN, 但加载的是 MLP 模型
- 修复: 从 checkpoint 的 key 名自动检测模型类型 (contain "conv" → CNN)

## 14. Git clone 中文路径失败
- 症状: `fatal: could not create leading directories`
- 原因: bash 工具编码问题
- 修复: 改用 PowerShell 或 WebFetch
